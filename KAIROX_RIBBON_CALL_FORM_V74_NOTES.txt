Kairox Ribbon Call/Form Fix v74

Fixed:
1. Phone icon gating:
   - Phone/ribbon call now always opens the lead form first.
   - If details already exist in the session, the form is prefilled.
   - Voice call starts only after the user submits the form.
   - Submitted details continue to pass to the Retell/n8n call webhook as separate fields:
     name, phone, email, company, sessionId.

2. Ribbon tooltips:
   - Removed floating Call/Chat label tooltips that could overlap the expand/collapse tab.
   - Kept accessible aria labels for screen readers.

3. Close/collapse state:
   - Closing the chat window now also syncs/collapses ribbon state.
   - Toggle uses actual DOM state to prevent the two-click expand issue.
   - Reduced toggle debounce for more responsive one-click expand/collapse.
